VALUE

三个核心价值

01

更快评估论文价值

先把论文拆成可实现要求、依赖与风险，再判断是否值得工程投入，避免盲目复现带来的沉没成本。

02

沉淀 Agent 多轮语料

每次运行自动产出多轮 Agent trajectory 语料（Anthropic tool_use / tool_result 结构化格式），并完整保留 LLM 与 MCP 原始调用记录——这是市面上其他复现项目不具备的能力。

03

避免低质量结果误交付

质量门禁会标记缺失文件、接口不一致、无法运行与机制未覆盖等问题，让失败成为可解释的诊断结论。

PROCESS

六阶段证据链

从 PDF 输入到验证报告，每一步都留下结构化数据，过程完全可回放。

01 论文解析 PDF 转写为 paper.md，提取原文段落与图表索引。
02 关键 claims 定位 从摘要和方法章节锁定可实现要点与公式。
03 老师傅批判 输出必须实现点、隐性陷阱与外部依赖清单。
04 实现计划 规划模块结构、文件路径与接口契约。
05 代码生成 工程化输出代码，全程记录事件流与 LLM 调用。
06 验证报告 结构化质量结论与差异诊断，给出明确通过/未通过。
